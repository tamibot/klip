INSTALL Klip for Mac · free & open source · macOS 14+

Klip is a tiny and fast Mac clipboard manager with screen capture, OCR, on-device voice-to-text and encrypted credentials. Built with love in Swift — no Electron, no accounts, no telemetry.

What's inside

Everything you copy while building with AI, one keystroke away.

⚡️

Designed to be lightweight

A native Swift menu-bar app. No Electron, no accounts, no background sync — your clips never leave your Mac.

🔎

Searchable history

Press ⌥⇧E and your last hundred clips are one keystroke away — instant search with highlighting, filters by type, full keyboard navigation.

✂️

Capture & annotate

Press ⌥⇧D, drag over the frozen screen and mark it up — arrows, text, highlighter, shapes, undo. It lands in history and on the clipboard.

🔤

Text recognition (OCR)

Came by a text that won't select? Press ⌥⇧F and select an area — Klip parses the text and copies it to the clipboard, with a little toast to confirm.

🎙️

Voice → text

Press ⌥⇧R, dictate, press again — the transcript lands on your clipboard. Runs fully on-device with Whisper; no API key, nothing uploaded.

🎬

Video → text

Drop a video (or any audio file) with ⌥⇧O — Klip extracts the audio track and transcribes it, with per-upload language.

🧹

Always paste clean

A copy from a dark-themed AI chat is stored as clean text — bold, italics and emojis kept; backgrounds, colours and fonts dropped.

💬

Copy for where it's going

One click reformats a clip for WhatsApp (*bold*, • bullets), rich email, Markdown, or a fenced code block (⌘↩) for AI chats.

🔑

Encrypted credentials

Tokens and API keys are auto-detected, shown masked, encrypted at rest (AES-256-GCM, key in the Keychain) and never auto-pasted.

⭐️

Favorites & names

Star the clips you reuse and give them names like “Deploy script” — filter to favorites and find things by what they do.

📦

Bundle context

Multi-select clips and combine them into a PDF or ZIP — upload a whole task's context to the AI in one shot.

🌍

8 languages

English, Spanish, French, German, Italian, Portuguese, Chinese and Japanese. OCR reads your system's languages too.

Tips and Tricks

Small things that make the loop faster.

Paste into an AI chat, cleanly

Select a clip and hit ⌘↩ — it's copied as a fenced code block with the language auto-detected. No more mangled indentation.

Back out with Esc

Esc exits multi-select first, then clears the search, then closes the panel — one key, layer by layer.

Star it, then filter

Hit the ★ on a clip and it joins your Favorites filter — right next to “All”. Stars don't reorder your history.

Keyboard all the way

↑/↓ to move, Enter to paste, ⌘⇧F to star, ⌘⌫ to delete. The mouse is optional.

Teach the transcriber your words

Add names like Klip, Supabase, webhook in Preferences › Context words — dictation stops writing “clip”.

Rename your keys

Detected credentials are masked and encrypted. Rename one to “Prod DB URL” and find it by name — it's never auto-pasted.

Install

Builds from source with just the Xcode Command Line Tools — no full Xcode.

# Command Line Tools (if you don't have them)
xcode-select --install

# Clone & install
git clone https://github.com/tamibot/klip.git klip
cd klip
./install.sh

The 📋 icon appears in the menu bar — press ⌥⇧E. Currently macOS only (14+, Intel or Apple Silicon); Windows is coming. A prebuilt download will land on GitHub releases.

FAQ

Is Klip free?

Yes — free and open source under MIT. Use it, fork it, ship it. If it helps you, a star on GitHub helps us.

Where do my clips and API keys live?

On your Mac, in ~/Library/Application Support/Klip with 0600 permissions. Credentials are AES-256-GCM encrypted at rest. Nothing is uploaded except the audio you explicitly send to a cloud transcriber — and on-device is the default.

When does Klip talk to the internet?

Only if you choose OpenAI/Gemini for transcription (your own key), or on the one-time Whisper model download. No telemetry, no update pings, no analytics. The code is public — check it.

Which macOS versions are supported?

macOS 14 (Sonoma) and up, Intel or Apple Silicon. On-device transcription is faster on Apple Silicon but works on both.
